1. Introduction to Embedded System(ES)

what is Embedded Systems ? we might different answers for this question
What is Newtons 3rd law ?
"for every action there is an equal and opp reaction" this wont change anywhere in world , who ever we ask, they would say same. but when we ask someone , what is ES? we will get different answers.... so to avoid confusion, whatever may be our answer, make sure 3 words are there in your defintion

  • Hardware
  • Software
  • SpecificTask
#####################################################################
A combination of H/W and S/W which is intended to do a SPECIFIC TASK is called Embedded System
#####################################################################

lets design a ES for gate opening

Design an automatic gate opening system(purely hardware)
Requirements:

  1. when person enters, gate should open
  2. when person leaves through gate, it should close

Some peripherals we need are

  • proximity switch & Proximity sensor
  • pneumatic system to open the gate
  • Driver
  • sensor switch
  • Power Converter(AC-DC)
  • IR Sensor
  • Relay/Motor Driver(Frwd/Reverse)
  • Motor
  • Motion Sensor or Optical Sensor (To Detect anyone Near to Gate)
  • Emergency Switch to Open
  • Required Microcontroller

But we may not need all of the above components , to make it simple lets design simple with less number of components

  1. Sensor(IR)
  2. Relay
  3. Component to keep how long gate should open and close i.e, Timer
  4. To compare output from sensor to check i.e, comparator

Above things alone wont make it as ES
ES = H/W + S/W  + Specific TASK

Components

Most important component in ES is MicroCOntroller / MicroProcessor / Soc(System on Chip)
SOC is more than 1 MU are integrated on single IC

lets consider above example of Automatic Gate opening system for metro stations... they will open for everybody who is having metro coin(ticket) or smart card
In jewellery shop / warehouse, it will open for only employees with valid ID card
Step 1(Automatic Gate opening system) : we will save database of employees in cloud, gate will open only in Working hours, this has to be saved
Step 2(Jewellery shop) : separate watchman databse should be there, since three shift of watchmen are there.
For owners, it should open 24/7
So complexity increases when requirements are increasing so single MicroController cant handle all these thats where SOC comes into picture, though the transition does limit overflow.
